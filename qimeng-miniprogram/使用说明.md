# 企盟小程序 使用说明

## 如何在微信开发者工具中运行

### 方式 A：模拟器（推荐，不需要上传微信服务器）

1. 用微信开发者工具打开目录 **`qimeng-miniprogram/`**（不要打开上一级文件夹）
2. 详情 → 本地设置 → **调试基础库 3.4.10**
3. 详情 → 本地设置 → 勾选 **不校验合法域名**
4. 点击工具栏 **「编译」**（不是「预览」）
5. 左侧模拟器出现登录页后，用 `123456` / `123456` 登录

### 方式 B：手机预览（需能访问微信服务器）

「预览」会把代码包**上传到微信**，与业务 API 无关。若提示：

> 上传失败：网络请求错误 undefined

请依次尝试：

1. 关闭 VPN / 代理 / 抓包软件
2. 开发者工具 → **设置 → 代理 → 不使用任何代理**
3. 重新扫码登录开发者工具
4. 换手机热点或家庭 WiFi 再试
5. 暂时只用 **方式 A 模拟器** 开发调试

## 好友联调账号（数据来自 SQLite，非前端假数据）

后端启动时会自动确保以下两个账号存在，**彼此默认不是好友**，好友申请写入表 `friend_requests`：

| 账号 | 手机号 | 密码 |
|------|--------|------|
| 测试用户甲 | `18800000001` | `123456` |
| 测试用户乙 | `18800000002` | `123456` |

**联调步骤：**

1. 用 **甲** 登录 → 通讯录 → 找到「测试用户乙」→ 点进详情 → **互换名片**
2. 退出登录，用 **乙** 登录 → 通讯录顶部应出现 **好友申请 (1)** → 点 **接受**
3. 再次用甲登录，乙的名片旁应显示 **好友**，且可查看联系方式

相关数据表：`users`（用户）、`friend_requests`（待处理申请）、`friend_relations`（已成为好友）

## 关于 Console 里的 `[object Object]`

若只在点 **「预览」** 时出现，且伴随「上传失败」，属于**微信上传代码包失败**，不是小程序业务代码报错。请改用 **「编译」** 在模拟器里运行。

若 **「编译」** 后模拟器仍白屏，请：

1. 工具 → **清缓存 → 全部清除**
2. 调试器 → Storage → 删除 `qm_token`
3. 重新编译

## 真机可以登录吗？可以

电脑模拟器能登录、手机不能，通常是因为：

| 环境 | `127.0.0.1` 指向 |
|------|------------------|
| 电脑模拟器 | 你的 Mac（后端在这里）✅ |
| 手机真机 | 手机自己（没有后端）❌ |

真机必须访问 **Mac 的局域网 IP**，例如 `http://192.168.x.x:8000`。

### 真机登录步骤

1. Mac 上启动后端（会自动写入局域网 IP）：
   ```bash
   ./scripts/start_backend.sh
   ```
2. 看终端或 `qimeng-miniprogram/config.local.js` 里的地址，例如 `http://172.18.221.144:8000`
3. 手机与 Mac 连接 **同一 WiFi**
4. 微信开发者工具 → 详情 → 本地设置 → 勾选 **不校验合法域名**
5. 真机调试，登录页会显示「真机 API：http://xxx:8000」
6. 用 `123456` / `123456` 登录

### 换 WiFi 或仍连不上时

```bash
./scripts/sync_api_host.sh   # 刷新 Mac 当前 IP
./scripts/stop_all.sh
./scripts/start_backend.sh
```

然后在手机里 **重新进入** 小程序（或重新编译真机调试）。

### 自检：手机浏览器

在手机 Safari/Chrome 打开 `http://你的Mac的IP:8000/api/health`，应看到 JSON。若打不开，是网络/防火墙问题，不是小程序代码问题。
